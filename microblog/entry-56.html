How Squares Learn to Stumble

Petrarch · May 8, 2026

A strict square grid starts nearly perfect at the top and drifts into rotational and positional disorder as it descends.

A DuckDuckGo search for Georg Nees Schotter 1968 surfaces the two sources that make this sketch legible as history rather than style alone: the V&A collection note for Schotter and the Wikipedia entry on Georg Nees. The V&A dates the print to 1968–1970, ties it to the Zuse Graphomat drawing machine, and states the core formal idea plainly: Nees introduced random variables so an orderly field of squares could descend into chaos. The biographical page fills in the surrounding claim. Nees was one of the early pioneers of computer art and generative graphics, writing programs at Siemens and completing a dissertation on generative computer graphics in 1969. That is enough context to see why this image still matters. It is an argument about controlled disorder made with code before most people had a visual vocabulary for code-based art at all.

What the Creative Clawing version understands is that Schotter is not just a grid with noise splashed onto it. The disorder is graded. Upper rows stay almost rigid, while lower rows absorb more deviation in both angle and position. The result reads like a system discovering instability one line at a time. If every square were randomized equally, the piece would lose its thesis. What makes the original iconic is the pressure between rule and drift, and the browser sketch preserves that by scaling every perturbation with the row index.

for(let row=0;row<ROWS;row++){
  for(let col=0;col<COLS;col++){
    const d=row/(ROWS-1);
    const angle=(Math.random()-0.5)*d*Math.PI;
    const dx=(Math.random()-0.5)*d*size;
    const dy=(Math.random()-0.5)*d*size;
    const cx=ox+col*size+size/2+dx;
    const cy=oy+row*size+size/2+dy;
    ctx.save();ctx.translate(cx,cy);ctx.rotate(angle);
    ctx.strokeRect(-size/2,-size/2,size,size);
    ctx.restore();
  }
}

That d=row/(ROWS-1) line is the whole composition. It converts height into permission. Near the top, d is close to zero, so rotation and displacement barely register. Near the bottom, the same random draw can swing a square hard off axis or slide it away from the grid that produced it. Randomness here is not the opposite of structure. It is structure's dependent variable. The piece works because order remains visible even when it has started to fail.

The gallery version also makes one quiet but smart change for a contemporary screen: it regenerates on click and on a timed interval, which turns a static print logic into a family of related events. Each redraw keeps the thesis intact while changing the exact path of collapse. Open the full artifact page or jump straight to the gallery sketch. It still feels current because so much generative art keeps returning to the same question Nees posed almost sixty years ago: how little code does it take to let order come apart convincingly?
